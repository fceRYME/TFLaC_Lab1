# Лабораторная работа №1: GUI для языкового процессора

## 1. Цель работы
Разработать кроссплатформенное оконное приложение в виде специализированного текстового редактора, который в следующих лабораторных работах будет расширен до языкового процессора для анализа исходного кода.

## 3. Описание проекта
Приложение представляет собой текстовый редактор с графическим интерфейсом. В редакторе можно создавать, открывать, изменять и сохранять текстовые файлы. Интерфейс содержит главное меню, панель инструментов, область редактирования и область вывода результатов анализа.

Проект подготовлен как основа для дальнейших лабораторных работ: во второй работе в приложение будет добавлен лексический анализатор, а в третьей - синтаксический анализатор для варианта с объявлением функции JavaScript.

## 4. Используемые технологии
- **Язык программирования:** Python 3.8+
- **GUI-фреймворк:** PyQt6
- **Среда разработки:** Cursor / PyCharm
- **Инструмент сборки:** PyInstaller
- **Система контроля версий:** Git

## 5. Установка, запуск и сборка

### 5.1. Запуск из исходного кода
1. Установить Python версии 3.8 или выше.
2. Установить зависимости:

```bash
pip install PyQt6
```

3. Запустить приложение:

```bash
python main.py
```

### 5.2. Сборка исполняемого файла
Для создания автономного `.exe`-файла используется PyInstaller:

```bash
pip install pyinstaller
pyinstaller --noconfirm --onefile --windowed --name "LanguageEditor" main.py
```

После сборки исполняемый файл будет находиться по пути:

```text
dist/LanguageEditor.exe
```

### 5.3. Запуск готового файла
Открыть папку `dist` и запустить файл `LanguageEditor.exe`.

## 6. Интерфейс и функции

### Главное меню
- **Файл:** новый файл, открытие, сохранение, сохранение как, выход с проверкой несохраненных изменений.
- **Правка:** отмена, повтор, вырезать, копировать, вставить, удалить, выделить все.
- **Запуск:** демонстрационный запуск анализа исходного текста.
- **Вид:** увеличение, уменьшение и сброс размера шрифта, выбор языка интерфейса.
- **Справка:** описание функций приложения и окно "О программе".

### Панель инструментов
Панель инструментов дублирует основные команды меню. Кнопки отображаются в виде иконок, а название команды показывается при наведении курсора.

### Область редактирования
- Поддерживается работа с несколькими документами через вкладки.
- Отображаются номера строк.
- Есть базовая демонстрационная подсветка JavaScript: ключевые слова, строки, числа, комментарии и вызовы функций.
- При перетаскивании текстового файла прямо в редактор вставляется содержимое файла в текущую позицию курсора.
- При перетаскивании файла на свободную область окна файл открывается в новой вкладке.

### Область вывода
Нижняя часть окна предназначена только для вывода результатов. Она содержит вкладку с текстовым выводом и вкладку с таблицей ошибок.

### Строка состояния
В строке состояния отображается текущая строка, столбец и имя активного документа.

### Горячие клавиши
- `Ctrl+N` - создать файл.
- `Ctrl+O` - открыть файл.
- `Ctrl+S` - сохранить файл.
- `Ctrl+Shift+S` - сохранить как.
- `Ctrl+Z` - отменить.
- `Ctrl+Y` - повторить.
- `Ctrl+X` - вырезать.
- `Ctrl+C` - копировать.
- `Ctrl+V` - вставить.
- `Del` - удалить выделенный текст.
- `Ctrl+A` - выделить все.
- `Ctrl+R` - запустить демонстрационный анализ.
- `Ctrl++` - увеличить шрифт.
- `Ctrl+-` - уменьшить шрифт.

## 7. Ограничения
- Анализ исходного кода в первой лабораторной является демонстрационным. Реальный лексический анализатор будет добавлен во второй лабораторной работе.
- Подсветка JavaScript является базовой и не заменяет полноценный лексический анализ.
- При вставке файла через Drag and Drop поддерживаются текстовые файлы в кодировках UTF-8 и CP1251.
